# CI/CD And Releases

## Workflows

### `ci.yml`

Runs on pull requests, pushes to `main`, and manual dispatch.

Validation steps:

- installs Node.js and pnpm
- installs required Linux build dependencies
- runs `pnpm install --frozen-lockfile` in `codex-linux-build/`
- checks shell syntax through `pnpm run verify`
- runs `shellcheck` on the main shell entrypoints
- generates sample release notes from the current Git history

Smoke build:

- runs on non-PR events
- builds the portable Linux artifact
- uploads the resulting archive, checksum, and build metadata as workflow artifacts
- runs a second smoke job that converts the portable archive into an Arch `pkg.tar.zst` inside an `archlinux` container

### `release.yml`

Runs on tag pushes matching `v*`.

Release steps:

- installs the same build toolchain as CI
- builds the portable artifact with `RELEASE_TAG=$GITHUB_REF_NAME`
- turns that artifact into a pacman package via `scripts/build-arch-package.sh` and `packaging/arch/PKGBUILD`
- generates release notes from commits between the previous tag and the current tag
- creates or updates the GitHub Release
- uploads the portable archive, the Arch package, checksums, and metadata

## Release Notes Strategy

Release notes are generated by [`scripts/generate-release-notes.sh`](../scripts/generate-release-notes.sh).

The script includes:

- current tag
- release date
- previous tag if one exists
- commit range between tags
- upstream Codex version embedded in the built artifact when available
- every commit subject in the range
- every commit body in the range as the "comment" section

That means a pushed tag automatically gets a release body containing the full commit commentary for the iteration.

## Local Commands

Install dependencies:

```bash
cd codex-linux-build
pnpm install --frozen-lockfile
```

Fast validation:

```bash
pnpm run verify
shellcheck build.sh start.sh ../scripts/generate-release-notes.sh ../scripts/build-arch-package.sh ../packaging/arch/codex-desktop-wrapper.sh
```

Portable package:

```bash
pnpm run package:portable
```

Arch package from the portable artifact:

```bash
./scripts/build-arch-package.sh \
  --source codex-linux-build/artifacts/*.tar.gz \
  --metadata codex-linux-build/artifacts/build-metadata.env \
  --output-dir codex-linux-build/artifacts
```

Manual release notes preview:

```bash
cd ..
./scripts/generate-release-notes.sh --ref v1.0.0
```

## Artifact Contract

Each packaged release contains:

- `dist/` adapted upstream application bundle
- `start.sh` portable launcher
- `node_modules/` runtime dependencies installed from the locked tool package
- `build-metadata.env` build metadata
- `codex-icon.png` launcher icon

GitHub Release uploads:

- `codex-desktop-<tag>-linux-x64.tar.gz`
- `codex-desktop-<tag>-linux-x64.tar.gz.sha256`
- `codex-desktop-bin-<pkgver>-1-x86_64.pkg.tar.zst`
- `codex-desktop-bin-<pkgver>-1-x86_64.pkg.tar.zst.sha256`
- `build-metadata.env`
- `release-notes.md`

## Operational Notes

- No extra GitHub secrets are required; the workflows use the default `GITHUB_TOKEN`.
- The release flow is idempotent for the same tag: rerunning it updates the existing release and replaces uploaded files.
- Because the build depends on the official upstream DMG, network access to OpenAI's download host is required during smoke builds and releases.
